基础 CSS Sprites：background-position 切图

将多个小图标合并为一张大图，通过 background-position 在不同位置显示不同图标。 本演示使用内联 SVG data URI 模拟雪碧图，与真实 PNG 雪碧图的机制完全一致。

1 雪碧图全貌

下方是一张 256×128 的"雪碧图"，包含 4 列 × 2 行共 8 个图标单元，每单元 64×64 像素。 第 1 行是默认状态，第 2 行是 hover 状态（背景更亮）。

整张雪碧图尺寸：256 × 128 px  |  单元格：64 × 64 px  |  布局：4 列 × 2 行

左上角坐标 (0, 0)，向右 X 增大，向下 Y 增大。 background-position 用负值把目标单元"推入" 64×64 视窗。

关键点： 整张雪碧图只需要 1 次 HTTP 请求 即可下载。 页面上无论出现多少个图标，浏览器都从同一张已缓存的图中"截取"对应区域， 不会产生额外请求。

2 图标网格（同一张图，不同 position）

下面 4 个图标全部使用 .sprite 基类（同一张背景图）， 仅 background-position 不同。把鼠标移上去可以看到 hover 效果。

Home
background-position: 0 0
User
background-position: -64px 0
Star
background-position: -128px 0
Heart
background-position: -192px 0
注意： 每个图标的 HTML 都是同一个 <span class="sprite icon-xxx"></span>， 浏览器只下载一次雪碧图，4 个图标复用同一份资源。

3 background-position 原理可视化

左侧是"视窗 + 完整雪碧图"模型：黄色框是 64×64 的固定视窗（即元素可见区域）， 下方移动的是完整雪碧图。点击右侧按钮，观察背景图如何被"推"到正确位置， 使目标单元落入视窗。

▼ 可视化模型（背景图在视窗下移动）

当前 background-position: 0 0

▼ 实际显示效果（64×64 元素）
原理： background-position: -64px 0 表示把背景图向左移动 64px， 使原图中横坐标 64~128 的区域（第 2 个图标）对齐到视窗的 0~64 区域。 负号 = 背景图向左/上移动；正值 = 向右/下移动。

4 悬停切换状态（零额外请求）

把鼠标移到下面的图标上。hover 时切换到雪碧图第 2 行（更亮的版本）， 由于整张图早已下载，不会产生任何闪烁或额外请求。

← 鼠标悬停观察颜色变化（默认态 ↔ hover 态）→

.icon-home:hover
.icon-user:hover
.icon-star:hover
.icon-heart:hover
对比传统方式： 如果每个图标用单独的 <img>，hover 时改 src 会触发新请求，出现"白闪"。雪碧图把两种状态放在同一张图里， 切换 background-position 是瞬时完成的。

5 代码示例

以下是本演示的核心 CSS 与 HTML，可直接复制使用。

CSS 
/* 1. 基类：所有图标共用同一张雪碧图 */
.sprite {
  width: 64px;
  height: 64px;
  background-image: url("./sprite.png");  /* 真实项目用 PNG */
  background-repeat: no-repeat;          /* 禁止平铺 */
  display: inline-block;
}

/* 2. 每个图标只改 background-position */
.icon-home  { background-position: 0      0;    }  /* 第 1 列 */
.icon-user  { background-position: -64px  0;    }  /* 第 2 列 */
.icon-star  { background-position: -128px 0;    }  /* 第 3 列 */
.icon-heart { background-position: -192px 0;    }  /* 第 4 列 */

/* 3. hover 切到第 2 行（y = -64px），无额外请求 */
.icon-home:hover  { background-position: 0      -64px; }
.icon-user:hover  { background-position: -64px  -64px; }
.icon-star:hover  { background-position: -128px -64px; }
.icon-heart:hover { background-position: -192px -64px; }
HTML 
<!-- 每个图标只是同一个 span，类名不同 -->
<span class="sprite icon-home"></span>
<span class="sprite icon-user"></span>
<span class="sprite icon-star"></span>
<span class="sprite icon-heart"></span>

<!-- 真实场景：导航栏图标 -->
<nav>
  <a href="/"><span class="sprite icon-home"></span> 首页</a>
  <a href="/user"><span class="sprite icon-user"></span> 我的</a>
</nav>
Retina 适配（@2x） 
/* 高分屏使用 2 倍图，再缩小到 1 倍尺寸 */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) {
  .sprite {
    background-image: url("./sprite@2x.png");
    background-size: 256px 128px;  /* 缩到 1 倍逻辑尺寸 */
  }
}

6 请求对比：传统多图 vs 雪碧图

假设页面上有 4 个图标，对比两种方式的网络请求差异（HTTP/1.1 场景）。

✗ 传统方式：每个图标一张图

4 次请求
  • home.png — 1.2 KB
  • user.png — 1.4 KB
  • star.png — 1.1 KB
  • heart.png — 1.3 KB
  • 总计 ≈ 5.0 KB + 4 次 TCP/TLS 握手开销
  • hover 还需额外请求 hover 版本（共 8 次）

✓ CSS Sprites：合并为一张图

1 次请求
  • sprite.png — 3.8 KB（合并后更小）
  • 1 次 TCP/TLS 握手
  • hover 状态已内置，0 次额外请求
  • 浏览器缓存命中率更高
HTTP/2 时代注意： HTTP/2 支持多路复用，多个请求可共用一个连接，"减少请求数"的收益已大幅下降。 在 HTTP/2 环境下，雪碧图不再是必须，应根据实际场景评估。
